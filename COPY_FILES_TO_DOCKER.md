# Копирование существующих файлов в Docker

## Проблема
У вас уже есть файлы в локальной директории `uploads/`, но они не видны в Docker контейнере.

## Решение 1: Монтирование локальной директории (уже применено)
Теперь `docker-compose.yml` монтирует локальную директорию `./uploads` в контейнер.
Это означает, что:
- ✅ Файлы из `uploads/` сразу видны в контейнере
- ✅ Изменения синхронизируются в обе стороны
- ✅ Файлы не теряются при перезапуске

## Решение 2: Если нужно использовать volume вместо монтирования

Если вы хотите использовать Docker volume (для продакшена), нужно скопировать файлы:

### Шаг 1: Верните volume в docker-compose.yml
```yaml
volumes:
  - app_uploads:/opt/app/uploads
```

### Шаг 2: Скопируйте файлы в volume
```bash
# Остановите контейнер
docker-compose down

# Скопируйте файлы из локальной директории в volume
docker run --rm -v app_uploads:/uploads -v $(pwd)/uploads:/source alpine sh -c "cp -r /source/* /uploads/ 2>/dev/null || true"

# Запустите контейнер
docker-compose up -d
```

### Шаг 3: Проверьте
```bash
docker run --rm -v app_uploads:/uploads alpine ls -la /uploads/audio/
```

## Текущая конфигурация (рекомендуется для разработки)

Сейчас используется монтирование локальной директории, что удобно для разработки:
- Файлы доступны сразу
- Не нужно копировать
- Изменения видны сразу

## Для продакшена

В продакшене лучше использовать volume и загружать файлы через API или копировать их в volume при деплое.
